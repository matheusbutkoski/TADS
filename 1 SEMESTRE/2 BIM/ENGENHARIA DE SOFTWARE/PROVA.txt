TESTE DE SOFTWARE
TEM COMO OBJETIVO REVELAR FALHAS E BUGS PARA QUE SEJAM
CORRIGIDAS ANTES DO PRODUTO FINAL

PROCESSO DE TESTE DE SOFTWARE

ESTRUTURA AS ETAPAS, ATIVIDADES E OS PAPEIS DO TESTE,
PERMITINDO ORGANIZACAO E CONTROLE DO CICLO DE TESTE.

BASEIA-SE EM UMA METODOLOGIA ADERENTE AO PROCESSO DE DESENVOLVIMENTO

CONCEITOS

FASE DE TESTE DE SOFTWARE

TESTE DE UNIDADE(TESTE UNITÁRIO)
PRIMEIRO TESTE REALIZADO
TESTA UNIDADES DE UM CÓDIGO
ELABORADO PELO DESENVOLVEDOR
CAIXA-BRANCA

TESTE DE INTEGRAÇÃO
INTEGRA AS UNIDADES
VERIFICA SE AS UNIDADES EM COJUNTO TRABALHAM DE ACORDO COM O
QUE FOI PROJETADO
CAIXA-BRANCA

TESTE DE SISTEMA
SISTEMA TESTADO COMO UM TODO
SIMULA A EXPERIÊNCIA DO USUÁRIO
BASEADO NOS REQUISITOS

TESTE DE REGRESSÃO
REEXECUÇÃO PARA VERIFICAR SE AS MODIFICAÇÕES NAO PROPAGARAM
EFEITOS COLATERAIS INDESEJÁVEIS

TESTE DE ACEITAÇÃO
ONDE O PRODUTO É VALIDADO EM RELAÇÃO AS REQUISITOS
DEFINE SE O PRODUTO ESTÁ PRONTO PARA SER LANÇADO
ESTABELECER CONFIANÇA NO SISTEMA
ALFA E BETA TEST SÃO COMUNS NESSA FASE (FEITO PELOS CLIENTES
EM POTENCIAL)

TESTE DE MANUTENÇÃO
APLICADOS TESTES PARA VERIFICAR SE UMA ALTERAÇÃO NAO DANIFICOU
OUTRAS FUNÇÕES EXISTENETES 


TIPO DE TESTE:

MODELO DE ESTRATÉGIA DE TESTE
FUNCIONAIS
ESTRUTURAIS

CAIXA BRANCA

REALIZADO NO CODIGO FONTE
AVALIA A ESTRUTURA INTERNA
REALIZADOS EM TODOS OS NIVEIS
PERCORRE TODOS OS CAMINHOS POSSIVEIS


CAIXA PRETA

IGNORA O CODIGO FONTE 
PREOCUPA-SE EM COMO O SOFTWARE FUNCIONA
AVALIA O COMPORTAMENTO EXTERNO
ERROS DE INTERFACE, COMPORTAMENTO E DESEMPENHO
DIFICULDADE DE TESTAR TODAS AS ENTRADAS POSSIVEIS

TÉCNICAS DE TESTE:
TÉCNICA EMPREGADA NO PROCEDIMENTO DE TESTE


TESTE EXPLORATÓRIO
DEPENDE DA HABILIDADE E CONHECIMENTO DO TESTADOR
CAIXA PRETA
REALIZADO QUANDO NÃO HÁ MUITA DOCUMENTAÇÃO SOBRE O SISTEMA
DEFEITOS SÃO REPORTADOS


TESTE FUNCIONAL
CAIXA PRETA
ONDE DETERMINADA FUNCIONALIDADE É VERIFICADA SEM LEVAR EM
CONTA O CODIGO FONTE
ATESTA SE A APLICAÇÃO É CAPAZ DE DESEMPENHAR O QUE SE PROPÕE
A FAZER
UTILIZADO NA INTEGRAÇÃO, SISTEMA E ACEITAÇÃO

TESTE UNITÁRIO
VEIRICA O COMPORTAMENTO DAS MENORES UNIDADES EM PARTES ISOLADAS
DE UM SISTEMA
CAIXA BRANCA

TESTE DE PERFORMANCE
AVALIA A CAPACIDADE DE RESPOSTA, ROBUSTEZ, DISPONIBILIDAED
CONFIABILIDADE E ESCALABILIDADE
VERIFICA O DESEMPENHO EM ALTA CARGA DE TRABALHO

3 TIPOS

-CARGA
VERIFICA O DESEMPENHO QUANDO É SUBMETIDO A CARGAS VARIÁVEIS
DE USUÁRIOS OU TRANSAÇÕES

-CAPACIDADE
IDENTIFICA OS LIMITES DA APICAÇÃO

-TESTE DE ESTRESSE
VERIFICA A PERFORMANCE QUANDO ESTÁ SUBMETIDO A CARGAS NO
LIMITE OU ACIMA DO SUPORTADO


TESTE DE USABILIDADE
AVALIA E TEM COMO FOCO A EXPERIÊNCIA DO USUARIO
INDICA SOLUÇOES PARA UMA INTERFACE LIMPA E INTUITIVA

COBERTURA DE TESTES:

VISA VERIFICAR A EFICÁCIA DOS TESTES CONFORME OS REQUISITOS

VERIFICA SE OS CASOS DE TESTES COBREM OS REQUISITOS QUE ESTÃO
SENDO TESTADOS


TDD (TEST DRIVEN DEVELOPMENT)
SE BASEIA EM CICLOS ONDE PARA CADA FUNCIONALIDADE DO SISTEMA
UM TESTE É CRIADO ANTERIORMENTE

O TESTE INICIALMENTE FALHA E LOGO APÓS A IMPLEMENTAÇÃO DA 
FUNCIONALIDADE SERÁ RETESTADO
