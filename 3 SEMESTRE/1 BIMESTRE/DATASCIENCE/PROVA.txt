PROCESSAMENTO PARALELO:
MÉTODO QUE PERMITE QUE DOIS OU MAIS PROCESSADORES TRABALHEM EM PARTES DIFERENTES DE UMA DETERMINADA
TAREFA, VISANDO RESULTADOS MAIS RÁPIDOS, DIVIDE UM TAREFA EM VARIAS SUB TAREFAS PARA EXECUTA-LAS
EM PARALELO

HADOOP É UM FRAMEWORK DE PROCESSAMENTO PARELELO, AS SUB TAREFAS SÃO LEVADAS PARA O PROCESSADOR
DA MAQUINA DO CLUSTER, AUMENTANDO O VELOCIDADE DO PROCESSAMENTO DE GRANDES DADOS

ARQUIVOS SAO DIVIDIDOS EM BLOCOS DE 64 OU 128MB E COLOCADOS NOS NODES

NORMALIZAÇÃO DE DADOS:
MUDAR OS VALORES DAS COLUNAS NUMÉRICAS PARA USAR UMA ESCALA COMUM
	FORMULA MIN MAX: N = X - MIN/(MAX - MIN)

PADRONIZAÇÃO DE DADOS:
	FORMULA Z-SCORE: X - MEDIA/DESVPAD.
	
CLASSIFICAÇÃO E REGRESSÃO

NA CLASSIFICAÇÃO QUEREMOS DESCOBRIR OU DESCREVER A CLASSE DE UM FATO, JA NA REGRESSAO BUSCA-SE
PREVER A CLASSE DE FORMA NUMÉRICA
	EXEMPLO: VAI FAZER FRIO OU CALOR? - CLASSIFICAÇÃO 
	QUAL SERÁ A TEMPERATURA? - REGRESSÃO
	
REGRESSÃO LINEAR SIMPLES E MÚLTIPLA

SIMPLES: DUAS VARIÁVEIS
MÚLTIPLA: ENVOLVE MAIS VARIÁVEIS

REGRESSÕES PODEM SER POSITIVAS OU NEGATIVAS

POSITIVA SE UMA VARIAVEL AUMENTAR E A OUTRA TAMBEM
NEGATIVA SE UMA VARIAVEL AUMENTA E A OUTRA DIMINIU

EXEMPLO: VENDA DE CASACOS CONFORME TEMPERATURA = NEGATIVA
QUANTO MENOR A TEMPERATURA, MAIOR A VENDA DE CASACOS, LOGO REGRESSÃO SIMPLES NEGATIVA

MODELOS
NO MACHINE LEARNING, UM ALGORITMO APRENDE COM DADOS HISTÓRICOS, O APRENDIZADO
CONSTRÓI O QUE CHAMAMOS DE MODELO.

PARA TESTARMOS UM MODELO EXISTEM BASICAMENTE DOIS MÉTODOS:

HOLD OUT: DIVIDE DE FORMA ALEATÓRIA OS DADOS EM 2 GRUPOS, EM QUE UM SERÁ
UTILIZADO PARA TREINAMENTO DA MÁQUINA E O OUTRO SERÁ UTILIZADO PARA TESTAR
O MODELO, NORMALMENTE AS PROPORÇÕES UTILIZADAS SÃO 70/30 OU 80/20

VALIDAÇÃO CRUZADA: O DATASET É DIVIDIDO ALEATORIAMENTE EM 'K' GRUPOS E APENAS
1 SERÁ UTILIZO PARA TESTE, ENQUANTO TODOS OS OUTROS SERÃO UTILIZADOS PARA TREINAMENTO


DE ACORDO COM A QUANTIDADE DE INSTÂCIAS UM MODELO PODE OU NÃO FUNCIONAR BEM

-SUPERAJUSTE(OVERFITTING) QUANDO O MODELO APRENDE DETALHES DEMAIS
-SOBREAJUSTE(UNDERFITTING) QUANDO O MODELO POSSUI BAIXA TAIXA DE ACERTO, POIS OS EXEMPLOS DE
TREINAMENTO FORAM POUCO REPRESENTATIVOS OU O MODELO USADO É MUITO SIMPLES

PROBLEMA CLASSE RARA
QUANDO POSSUIMOS UMA CLASSE EM UMA RELAÇÃO QUE POSSUI POUCAS INSTÂCIAS COMPARADO COM A/AS OUTRAS
	EXEMPLO: APROVAÇÃO E REPROVAÇÃO DE ALUNOS, COMO NÃO EXISTEM MUITOS DADOS SOBRE A REPROVAÇÃO, 
	ATRIBUI-SE O PROBLEMA DA CLASSE RARA
	
MATRIZ DE CONFUSÃO
EXIBE A DISTRIBUIÇÃO DOS REGISTROS EM TERMOS DE SUAS CLASSES ATUAIS E SUAS CLASSES PREVISTAS
POSSUI
 VERDADEIROS POSITIVOS(VP): OBSERVAÇÕES QUE O VALOR PREVISTO PELO MODELO COMO POSITIVO É REALMENTE POSITIVO
 VERDADEIROS NEGATIVOS(VN): OBSERVAÇÕES QUE O VALOR PREVISTO PELO MODELO COMO NEGATIVO É REALMENTE NEGATIVO
 
 FALSOS POSITIVOS(FP): CASOS EM QUE O RESULTADO PREVISTO COMO POSITIVO PELO MODELO É NA VERDADE NEGATIVO
 FALSOS NEGATIVOS(FN): CASOS EM QUE O RESULTADO PREVISTO COMO NEGATIVO PELO MODELO É NA VERDADE POSITIVO


MÉTRICAS DE AVALIAÇÃO DO MODELO (ATRAVÉS DA MATRIZ DE CONFUSÃO)

ACURÁCIA: VP + VN / TOTAL DE INSTÂNCIAS

PRECISÃO: VP / (VP + FP)

RECALL: VP / (VP + FN)

A PRECISÃO TEM O INTUITO DE ENTENDER A PROPORÇÃO DE ACERTOS COM AS INSTANCIAS PREVISTAS COMO POSITIVAS PELO MODELO
JA O RECALL TEM O INTUITO DE ENTEDER A PROPORÇÃO DE ACERTO DAS INSTÂNCIAS REAIS POSITIVAS DO MODELO

EXEMPLO: 
	SE O MODELO PREVIU 10 CASOS COMO POSITIVOS E APENAS 5 DESSES FORAM REAIS, A PRECISÃO SE TORNA 50% (5 / 5 + 5)
	SE NA REALIDADE EXISTEM 20 CASOS POSITIVOS E COMO MODELO ACERTOU APENAS 5, O RECALL SE TORNA 25% S(5/ 5 + 15)

F1-SCORE: 2 * PRECISAO * RECALL / PRECISAO + RECALL
